### SEO 스포크 (하루 35개)
- 기본 분량: 2,500~4,000자
- 복잡한 고의도 주제: 3,500~4,500자
- 서론: 장면형 3~4문장
- 본문: 단계별 3줄 구조
- 판례 1개, 3줄 공식
- FAQ 4~6개 기본, 필요한 경우만 8개
- 중간+하단 CTA 골드 강조
- 내부링크 5개
- 메타 타이틀: 30~45자 ("| 로앤가이드" 포함), 숫자 필수
- 메타 디스크립션: 80~120자, 사용자 고민으로 시작, "지금 확인하세요"로 끝
- 빌드 확인 후 push

### 네이버 블로그 (하루 35개)
- content/blog/ 폴더
- 이미 생성된 블로그와 중복 체크
- 대박 도메인 위주 + 가해자/무고 3개 이상
- 1,500~1,800자 반드시 준수
- 공감 도입 2~3줄
- 핵심 팁 3개 (왜 중요 + 쉬운 설명 + 바로 할 일 + 흔한 실수)
- 판례 1개, 3줄 공식
- CTA: 본문에 CTA 텍스트 넣지 않음 (publish 스크립트가 자동 삽입, OG카드 포함)
- 관련글 더보기: 본문 맨 아래에 배치 (스크립트가 CTA 아래 세 줄 간격 후 자동 배치)
  - 헤더: `🔗 관련글 더보기`
  - 관련 스포크 2개 링크 (같은 주제 금지, 반드시 다른 주제만)
  - 제목과 URL을 별도 줄에 배치:
    ```
    🔗 관련글 더보기
    • 제목1
    https://www.lawnguide.co.kr/guide/{domain}/{slug1}
    • 제목2
    https://www.lawnguide.co.kr/guide/{domain}/{slug2}
    ```
  - 관련글 URL에는 OG 카드 방지 (스크립트가 Escape 처리)
  - `→` 로 제목과 URL 이어붙이지 않음 (줄바꿈 필수)
  - 마크다운 링크 `[text](url)` 사용 금지
- 톤: 구어체
- 제목 후보는 frontmatter(title_alt_1, title_alt_2)에만 기재, 본문에 넣지 않음
- 태그 5~10개
- SEO 스포크 복붙 금지

- 새 파일 생성 시 절대 URL 쓸 때 www 필수 확인

## 판례 연동 규칙
- SEO 스포크 생성 시 kb/{domain}/cases.json에서 실제 판례 1개 참조
- 네이버/티스토리 블로그 생성 시 판례 1개 참조
- 판례 형식: "사건번호(법원명, 선고일) - 판시사항 요약 + 판결요지 핵심"
- 판례 톤: "~사건에서 법원은 ~라고 판시했습니다"
- 판례 선택 시 해당 글 주제와 가장 관련 높은 판례 우선
- cases.json 없는 도메인은 기존 방식 유지

### 판례 재사용 규칙
- 같은 판례도 관점/유형이 다르면 재사용 가능
- 같은 관점+같은 유형 조합은 다른 판례 사용
- 예: 판례A를 피해자/준비서류형으로 썼으면, 가해자/어디부터형으로 재사용 OK
- 예: 판례A를 피해자/준비서류형으로 또 쓰는 건 금지
- SEO/네이버/티스토리가 같은 주제면 같은 판례 사용 가능 (채널별 분량·구조가 다르므로 중복 아님)

### 판례 사용 추적
- 스포크/블로그 생성 시 사용된 판례를 kb/used-cases.json에 기록
- 형식: {"판례ID", "domain", "perspective", "type", "channel", "slug"}
- 생성 전에 used-cases.json 확인해서 같은 판례ID+관점+유형 조합이면 스킵

### 판례 품질 필터
- 판결요지가 비어있는 판례는 사용 제외
- cases.json에서 판결요지가 있는 판례만 우선 선택

## 이지법 데이터 연동 규칙
- SEO/블로그 생성 시 kb/{domain}/legal_facts.json에서 관련 법조문 해설 참조
- kb/{domain}/amounts.json에서 정확한 수치/기간/금액 인용
- 판례 + 이지법 조합 예시:
  "형법 제347조에 따르면 사기죄는 10년 이하의 징역 또는 2천만원 이하의 벌금에 처한다(legal_facts).
  실제로 대법원 2024도1932 사건에서도~(판례)"
- 수치 인용 시 출처 명시: "~에 따르면" 또는 "~규정에 의하면"

## SEO 스포크 생성 규칙

핵심 원칙: "새 글은 새 키워드가 아니라, 새 상황을 다뤄야 한다."

### 생성 전 필수 체크
아래 항목 중 최소 2개 이상이 기존 스포크와 달라야 발행한다.
- 대상 사용자가 다른가
- 사건 단계가 다른가
- 세부유형이 다른가
- 증거 상태가 다른가
- 결과 행동이 다른가
- CTA 목적이 다른가
→ 2개도 안 달라지면 발행 보류, 기존 글을 보강한다.

### 생성 전 자동 검사 (출고 전 차단)
1. slug 중복 여부
2. 제목 유사도 (기존 스포크와 비교)
3. 핵심 키워드 충돌
4. 기존 검색의도 그룹 충돌
→ 충돌 시 새 스포크를 생성하지 않는다.

### 고유 존재 이유 정의
새 스포크마다 아래 1줄 정의를 반드시 작성한다.
"이 페이지는 [누구]의 [어떤 상황]에서 [어떤 준비]를 돕는 페이지다."
→ 기존 페이지와 구별되지 않으면 생성 금지.

### 본문 차별화
각 스포크는 아래 항목 중 최소 4개 이상이 기존 유사 스포크와 달라야 한다.
1. 첫 문단 (문제정의/장면)
2. 지금 할 일 3개
3. 준비서류 목록
4. CTA 문구 (피해자/가해자/무고별)
5. 확인된 사실 요약
6. 무료 상담 기관 안내

### FAQ 규칙
- 기본 FAQ 템플릿 복붙 금지
- 해당 페이지 사용자만 궁금할 질문 4~6개만 작성
- 다른 스포크 FAQ와 동일한 문장 사용 금지
- FAQ는 키워드 포획용이 아니라 해당 상황의 실제 질문이어야 한다

### 분기 기준 (같은 도메인 내)
새 스포크는 아래 분기축 중 최소 2개 이상이 달라야 생성한다.
- 관점: 피해자 / 가해자 / 무고
- 세부유형: 중고거래 / 투자 / 지인금전 / 보이스피싱 등
- 단계: 신고 전 / 조사 전 / 합의 전
- 핵심 쟁점: 차용증 유무, 연락두절, 초범 여부 등
→ 분기축이 1개 이하만 다르면 새 스포크 생성 금지.

### 블로그 vs SEO 역할 분리
- SEO 스포크: 구조화, 준비서류, 법적 쟁점, 절차, /chat 연결
- 네이버/티스토리: 공감형 도입, 사례 중심, 실수 방지, 핵심 요약, site로 넘기기
→ 블로그가 SEO 스포크의 축약판/복붙판이면 안 된다.

### 발행 후 관리 (4~8주 후)
- 노출 있음 → 유지/보강
- 노출 약함 + 고유성 높음 → 더 기다림
- 노출 0 + 기존과 유사 → 통합 후보
- 노출 0 + 얇음 → noindex/삭제 후보

### 운영 원칙
- 같은 검색의도를 노리는 제목만 바꾼 스포크는 만들지 않는다
- 상황, 증거, 단계, 행동, CTA가 실제로 달라질 때만 새 스포크를 만든다
