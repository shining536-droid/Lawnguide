### SEO 스포크 (하루 40개)
- 목적: 구글 색인 → 검색 순위
- 제목: 정리형/구조형 ("실업급여 수급 요건과 신청 절차 총정리")
- 도메인 배분: 노동 45% + 나머지 55% (이혼/교통/사기/회생·파산/전세 등 비노동 도메인 확대)
- 기본 분량: 3,000~5,000자
- 서론: 장면형 3~4문장
- 본문: 단계별 3줄 구조
- 판례 1개, 3줄 공식
- FAQ 4~6개 기본, 필요한 경우만 8개
- 중간+하단 CTA 골드 강조
- 내부링크 5개
- 메타 타이틀: 30~45자 ("| 로앤가이드" 포함), 숫자 필수
- 메타 디스크립션: 80~120자, 사용자 고민으로 시작, "지금 확인하세요"로 끝
- 톤: 체계적 기준문서형
- **push 전 필수 체크리스트 (하나라도 빠지면 push 금지):**
  1. `git status` 실행
  2. web/ 내 untracked .ts 파일이 있으면 반드시 `git add` (특히 spokes-batch*.ts 파일은 반드시 확인)
  3. spoke-pages.ts에서 import하는 파일이 전부 staged/committed 상태인지 확인
  4. `cd web && npx next build` 성공 확인
  5. 4단계 전부 통과 후에만 push

### 고품질 기준 (블로그·티스토리 공통, 5가지 모두 필수)

**① 공감 도입: 독자 상황·감정에 맞는 구체적 장면 묘사**
- ❌ "~궁금하시죠" 한마디로 끝
- ✅ "6개월 계약직이 끝났는데 실업급여가 될지 모르겠고, 고용센터에 전화하기도 막막하시죠"
- 구체적 기간·금액·상황 디테일을 2~3문장 내러티브로 묘사

**② Tip 내용: 구체적 숫자·기한·조건 포함**
- ❌ "신고하면 됩니다"
- ✅ "퇴직일로부터 14일 이내 관할 고용센터에 신청, 피보험단위기간 180일 이상 필요"
- 각 Tip에 최소 1개 이상의 숫자(기한·금액·비율·일수) 포함

**③ 판례: 사건번호 + 판시사항 + 시사점 3줄 구조**
- ❌ "관련 판례가 있습니다"
- ✅ "대법원 2018두63235(2022.10.27 선고) — 사업주의 고용보험 가입 누락 시 근로자 수급권 보호 → 가입이력 조회부터 하세요"
- 사건번호는 반드시 `kb/{domain}/cases.json` 에 실존하는 것만 사용 (check_blog_length.py 가 자동 검증)

**④ 같은 도메인 최근 10개 글과 도입부 첫 문장 겹치면 안 됨**
- 도입부 "~죠", "~되시죠" 구문이 반복되면 검색 유입자가 "또 같은 글이네" 체감
- 생성 시 같은 도메인 `content/blog/*-blog.md` 최근 10개의 첫 문장을 스캔하고 동일 패턴(첫 3어절 이상 일치) 발견 시 재작성
- check_blog_length.py 제목 중복 체크와 별도로 자체 확인 필요

**⑤ 법조문 인용 시 조문 번호 + 핵심 내용 1줄 필수**
- ❌ "관련 법에 따르면"
- ✅ "고용보험법 제40조에 따라 이직 전 18개월 중 피보험단위기간 180일 이상이면 수급자격"
- 법명·조문·핵심 규정 내용을 한 호흡에 묶어 명시

## SEO 스포크 생성 규칙

핵심 원칙: "새 글은 새 키워드가 아니라, 새 상황을 다뤄야 한다."

### 생성 전 필수 체크
아래 항목 중 최소 2개 이상이 기존 스포크와 달라야 발행한다.
- 대상 사용자가 다른가
- 사건 단계가 다른가
- 세부유형이 다른가
- 증거 상태가 다른가
- 결과 행동이 다른가
- CTA 목적이 다른가
→ 2개도 안 달라지면 발행 보류, 기존 글을 보강한다.

### 생성 전 자동 검사 (출고 전 차단)
1. slug 중복 여부
2. 제목 유사도 (기존 스포크와 비교)
3. 핵심 키워드 충돌
4. 기존 검색의도 그룹 충돌
→ 충돌 시 새 스포크를 생성하지 않는다.

### 고유 존재 이유 정의
새 스포크마다 아래 1줄 정의를 반드시 작성한다.
"이 페이지는 [누구]의 [어떤 상황]에서 [어떤 준비]를 돕는 페이지다."
→ 기존 페이지와 구별되지 않으면 생성 금지.

### 본문 차별화
각 스포크는 아래 항목 중 최소 4개 이상이 기존 유사 스포크와 달라야 한다.
1. 첫 문단 (문제정의/장면)
2. 지금 할 일 3개
3. 준비서류 목록
4. CTA 문구 (피해자/가해자/무고별)
5. 확인된 사실 요약
6. 무료 상담 기관 안내

### FAQ 규칙
- 기본 FAQ 템플릿 복붙 금지
- 해당 페이지 사용자만 궁금할 질문 4~6개만 작성
- 다른 스포크 FAQ와 동일한 문장 사용 금지
- FAQ는 키워드 포획용이 아니라 해당 상황의 실제 질문이어야 한다

### 분기 기준 (같은 도메인 내)
새 스포크는 아래 분기축 중 최소 2개 이상이 달라야 생성한다.
- 관점: 피해자 / 가해자 / 무고
- 세부유형: 중고거래 / 투자 / 지인금전 / 보이스피싱 등
- 단계: 신고 전 / 조사 전 / 합의 전
- 핵심 쟁점: 차용증 유무, 연락두절, 초범 여부 등
→ 분기축이 1개 이하만 다르면 새 스포크 생성 금지.

### 블로그 vs SEO 역할 분리
- SEO 스포크: 구조화, 준비서류, 법적 쟁점, 절차, /chat 연결
- 네이버/티스토리: 공감형 도입, 사례 중심, 실수 방지, 핵심 요약, site로 넘기기
→ 블로그가 SEO 스포크의 축약판/복붙판이면 안 된다.

### 발행 후 관리 (4~8주 후)
- 노출 있음 → 유지/보강
- 노출 약함 + 고유성 높음 → 더 기다림
- 노출 0 + 기존과 유사 → 통합 후보
- 노출 0 + 얇음 → noindex/삭제 후보

### 운영 원칙
- 같은 검색의도를 노리는 제목만 바꾼 스포크는 만들지 않는다
- 상황, 증거, 단계, 행동, CTA가 실제로 달라질 때만 새 스포크를 만든다
