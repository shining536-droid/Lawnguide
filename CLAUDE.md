## 일일 작업 루틴

### SEO 스포크 (하루 30개)
- seo-keywords 매트릭스에서 미생성 high 키워드
- 이미 생성된 스포크와 중복 체크
- 대박 도메인 70%, 피해자50/가해자30/무고20
- 유형 골고루: 상황형, 절차형, 비교형, Q&A형, 체크리스트형, 수치기한형
- v10 고품질, 5,000~6,500자
- 서론: 장면형 3~4문장
- 본문: 단계별 3줄 구조
- 판례 1~2개, 3줄 공식
- FAQ 8개, 첫 2개 open
- 중간+하단 CTA 골드 강조
- 내부링크 5개
- 이전 차수와 서론/FAQ 겹치지 않게
- 빌드 확인 후 push

### 네이버 블로그 (하루 20개)
- content/blog/ 폴더
- 이미 생성된 블로그와 중복 체크
- 대박 도메인 위주 + 가해자/무고 3개 이상
- 1,500~1,800자 반드시 준수
- 공감 도입 2~3줄
- 핵심 팁 3개 (왜 중요 + 쉬운 설명 + 바로 할 일 + 흔한 실수)
- 판례 1개, 3줄 공식
- CTA: 로앤가이드에서 내 상황 정리해보기 lawnguide.co.kr
- 톤: 구어체
- 제목 후보 3개, 태그 5~10개
- SEO 스포크 복붙 금지

### 티스토리 블로그 (하루 10개)
- content/tistory/ 폴더
- 네이버 블로그와 같은 주제
- 제목: 같은 키워드, 다른 표현
- 서론: 2~3문장 완전 재작성
- 본문: 팁 순서 변경
- 70% 이상 다르게 재작성
- CTA: 로앤가이드에서 내 상황 정리해보기 https://lawnguide.co.kr
- 1,500~1,800자

### 발행 명령어
- 썸네일 생성: python generate_thumbnails_lawnguide.py
- 네이버 발행: python publish_lawnguide.py --all
- 티스토리 발행: python publish_tistory.py --limit 10

### 규칙
- 가해자: "혐의를 받고 있다면"
- 무고: "사실과 다르게 신고되었다면"
- 톤: "~확인해보세요" / "~정리해보세요"
- 커밋 메시지 한글로

## 판례 연동 규칙
- SEO 스포크 생성 시 kb/{domain}/cases.json에서 실제 판례 1개 참조
- 네이버/티스토리 블로그 생성 시 판례 1개 참조
- 판례 형식: "사건번호(법원명, 선고일) - 판시사항 요약 + 판결요지 핵심"
- 판례 톤: "~사건에서 법원은 ~라고 판시했습니다"
- 판례 선택 시 해당 글 주제와 가장 관련 높은 판례 우선
- cases.json 없는 도메인은 기존 방식 유지

### 판례 재사용 규칙
- 같은 판례도 관점/유형이 다르면 재사용 가능
- 같은 관점+같은 유형 조합은 다른 판례 사용
- 예: 판례A를 피해자/준비서류형으로 썼으면, 가해자/어디부터형으로 재사용 OK
- 예: 판례A를 피해자/준비서류형으로 또 쓰는 건 금지
- SEO/네이버/티스토리가 같은 주제면 같은 판례 사용 가능 (채널별 분량·구조가 다르므로 중복 아님)
